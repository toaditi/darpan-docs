title: "Docs development"
description: "Preview and validate Darpan documentation changes locally."
---

<Info>
  **Prerequisites**
  - Node.js 20.17.0 or higher
  - The Mintlify CLI
  - This repository cloned locally
</Info>

<Steps>
<Step title="Install the Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Preview locally">

Run the docs site from the repository root:

```bash
mint dev
```

Open `http://localhost:3000`.
</Step>

<Step title="Use another port when needed">

```bash
mint dev --port 3333
```
</Step>

<Step title="Check links and syntax">

```bash
mint broken-links
mint validate
```
</Step>
</Steps>

## Lightweight validation

When the Mintlify CLI is unavailable, validate the JSON files:

```bash
jq empty docs.json
jq empty api-reference/openapi.json
```

## Publishing

Mintlify deploys from the connected GitHub repository after changes are pushed to `main`. Keep commits small and prefer one docs intent per commit.
