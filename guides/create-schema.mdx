---
title: "Create a schema"
description: "Define a reusable source-data contract."
---

Schemas tell Darpan how to parse and compare source files or records. A good schema turns a source shape into stable fields that a run can reuse.

## When to create a schema

Create a schema when a workflow needs to parse a new file shape, compare a new record shape, or preserve a reusable data contract for repeated runs.

## Steps

1. Open **Ask Darpan** with `Cmd/Ctrl+K` or the floating **Ask Darpan** button.
2. Search for **Create Schema**.
3. Provide the schema name or identifier.
4. Select the source system when the workflow asks for it.
5. Add or confirm the fields Darpan should read.
6. Review field names, data types, and required markers.
7. Save the schema.

Search for **Open Schema Library** when you need to inspect or reuse an existing schema.

## Review checklist

- Field names match the source system or agreed normalized names.
- Required fields are marked intentionally.
- Numeric, date, and identifier fields use stable types.
- The schema name is specific enough to understand in saved run history.

<Tip>
Prefer one clear schema per source shape over reusing a vague schema that needs manual interpretation every time.
</Tip>
