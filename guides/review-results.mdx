---
title: "Review results"
description: "Interpret generated reconciliation output."
---

Results explain what the run found and what to do next. Review them as evidence, not just as a downloadable file.

## Review order

1. Open **Ask Darpan** with `Cmd/Ctrl+K` or the floating **Ask Darpan** button.
2. Search by run name or output file name.
3. Select **Open Result: ...**.
4. Confirm run metadata and input sources.
5. Review match and mismatch counts.
6. Open generated output.
7. Classify each difference as source data, schema, mapping, rule behavior, or expected variance.

## Common result categories

| Category | Meaning |
| --- | --- |
| Matched | Darpan found corresponding records with no relevant difference. |
| Missing in source | A record appears in one source but not the other. |
| Value mismatch | A matched record has field-level differences. |
| Processing error | The run could not parse, compare, or output part of the data. |

## Next actions

- Fix source data when the mismatch is real.
- Adjust schema or field selection when parsing is wrong.
- Adjust rules when business classification is wrong.
- Rerun after correction so the new output preserves evidence.
