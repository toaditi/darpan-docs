Results show what Darpan found during a reconciliation run.

## Review order

1. Open **Ask Darpan** with `Cmd/Ctrl+K` or the floating **Ask Darpan** button.
2. Search by the run name or output file name.
3. Select the **Open Result: ...** result when it appears.
4. Confirm the run metadata.
5. Check whether the expected input files or source records were used.
6. Review match counts and mismatch counts.
7. Open generated output.
8. Identify whether each difference is source data, schema, mapping, or expected business variance.

## Common result categories

| Category | Meaning |
| --- | --- |
| Matched | Darpan found corresponding records with no relevant difference. |
| Missing in source | A record appears in one source but not the other. |
| Value mismatch | A matched record has field-level differences. |
| Processing error | The run could not parse, compare, or output part of the data. |

## Next actions

- Fix source data when the mismatch is real.
- Adjust schema or field selection when parsing is wrong.
- Adjust rules when business classification is wrong.
- Re-run after the correction to preserve evidence.
