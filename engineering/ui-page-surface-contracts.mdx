---
title: "UI page surface contracts"
description: "Shared Darpan UI patterns for static and workflow pages."
---

Use this page before creating or migrating Darpan UI pages.

## Static surfaces

- Frame static pages with `StaticPageFrame`.
- Use `StaticPageSection` for section blocks.
- Use `static-page-record-grid` and `static-page-record-tile` for saved records.
- Use `static-page-module-grid` and `static-page-module-tile` for in-surface navigation.
- Use `static-page-list-toolbar` and `static-page-pager` for saved-list chrome.
- Use `StaticEditableTitle` for inline editable names.

## Workflow surfaces

- Frame workflow pages with `WorkflowPage`.
- Build steps with `WorkflowStepForm`.
- Default create-entry workflows to one question per step.
- Use `WorkflowShortcutChoiceCards` for keyed multiple-choice prompts.
- Use `workflow-form-grid`, `workflow-form-grid--two`, and `workflow-form-grid--compact` before adding page-local layout CSS.
- Keep action buttons on shared controls such as `AppSaveAction` and `AppSelect`.

## Migration rules

- Split settings pages into static saved-record surfaces and workflow create/edit surfaces when a page is doing both jobs.
- Prefer shared utility classes in `src/style.css` over page-local scoped CSS when a pattern recurs.
- Preserve route behavior and workflow-origin state when static pages launch workflows.
- Keep custom page behavior in `darpan-ui`.

## Validation

Add or update page tests for shared contract classes. Run focused UI tests before broader UI checks.
