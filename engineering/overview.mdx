The Backend engine section explains how Darpan works at runtime. This section explains how to change it safely: choose the right repository, preserve contracts, and verify the affected surface.

## Engineering rules

- Start from the user workflow, especially reconciliation run setup and review.
- Read the existing route, API, service, entity, and processing code before changing behavior.
- Keep custom UI in the Vue app and backend contracts in the Moqui component.
- Use Moqui XML services to keep backend contracts visible and reviewable.
- Use Spark and Drools where dataset comparison or configurable decisioning belongs on the backend.
- Verify the smallest meaningful surface before claiming broader confidence.
- Remove weak compatibility layers when a simpler contract is available.
