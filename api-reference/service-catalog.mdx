title: "Service catalog"
description: "How Darpan service documentation should grow."
---

The service catalog should be built from verified backend facade contracts.

## Initial categories

| Category | Example responsibility |
| --- | --- |
| Auth | Login, logout, session info, active context. |
| Settings | AI, SFTP, NetSuite, and saved connection records. |
| Schemas | Schema library, create, edit, and detail operations. |
| Reconciliation | Run creation, result retrieval, generated output, and history. |

## Page requirements

Each documented service should include:

- Service name
- Purpose
- Authentication requirement
- Required inputs
- Optional inputs
- Response shape
- Error behavior
- Example request
- Source file or facade reference

## Do not document

- Internal helper services that are not stable integration contracts.
- Secrets or private environment values.
- Behavior inferred from plans but not present in code.
