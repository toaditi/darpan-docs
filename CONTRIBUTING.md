# Darpan Documentation Maintenance

Darpan documentation is maintained by the Darpan team.

This repository is not intended as an external contribution surface. Product, API, backend, and engineering docs should be updated by the team member responsible for the related Darpan change.

## Maintenance rules

- Keep docs tied to current product behavior and verified backend contracts.
- Use official product terms in tutorials.
- Do not publish customer data, credentials, cookies, tokens, Postman environments, or private Linear details.
- Validate changes with `mint validate` and `mint broken-links` before publishing broad updates.
- Push approved documentation updates to `main` so Mintlify can deploy them.
