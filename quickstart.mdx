This quickstart introduces the default Darpan workflow. Exact screen names may differ by release, but the sequence should stay stable.

<Steps>
<Step title="Confirm your environment">

Use the environment your team has assigned for testing or pilot work.

- Local frontend: `http://localhost:5173`
- Local backend: `http://localhost:8080`
- JSON-RPC endpoint: `http://localhost:8080/rpc/json`

</Step>
<Step title="Sign in">

Open Darpan and sign in with the credentials provided for your pilot company or test environment.

If login fails, start with [login and session troubleshooting](/troubleshooting/login-and-session).

</Step>
<Step title="Configure required connections">

Create or confirm the connection records needed by your workflow.

- SFTP for file exchange
- NetSuite for ERP integration flows
- AI provider settings when schema generation or assisted mapping is enabled

</Step>
<Step title="Create or select a schema">

Use a schema to describe the file or record shape Darpan should parse. Keep names stable and human-readable so saved runs remain understandable later.

</Step>
<Step title="Run reconciliation">

Start a reconciliation run, provide the source files or source records requested by the workflow, and wait for generated output.

</Step>
<Step title="Review results">

Open the run result, review differences, and decide whether the issue is source data, mapping, schema selection, or expected business variance.

</Step>
</Steps>
