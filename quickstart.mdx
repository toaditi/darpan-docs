This quickstart introduces the default Darpan workflow using Ask Darpan navigation. Exact screen names may differ by release, but the search-first sequence should stay stable.

<Steps>
<Step title="Confirm your environment">

Use the environment your team has assigned for testing or pilot work.

- Local frontend: `http://localhost:5173`
- Local backend: `http://localhost:8080`
- JSON-RPC endpoint: `http://localhost:8080/rpc/json`

</Step>
<Step title="Sign in">

Open Darpan and sign in with the credentials provided for your pilot company or test environment.

If login fails, start with [login and session troubleshooting](/troubleshooting/login-and-session).

</Step>
<Step title="Open Ask Darpan">

After sign-in, use **Ask Darpan** as your primary way to move through the app.

- Press `Cmd/Ctrl+K`.
- Or click the floating **Ask Darpan** button.
- Type the task, record, or workflow you want.
- Use Up/Down to select a result and Enter to open it.

</Step>
<Step title="Configure required connections">

Create or confirm the connection records needed by your workflow. In **Ask Darpan**, search for terms such as `SFTP`, `NetSuite`, `API key`, `token`, or `model settings`.

- SFTP for file exchange
- NetSuite for ERP integration flows
- AI provider settings when schema generation or assisted mapping is enabled

</Step>
<Step title="Create or select a schema">

Search **Ask Darpan** for `create schema`, `generate schema`, `upload schema`, or `schema library`. Use a schema to describe the file or record shape Darpan should parse.

Keep names stable and human-readable so saved runs remain understandable later.

</Step>
<Step title="Run reconciliation">

Search **Ask Darpan** for `run reconciliation`, `compare files`, `compare data`, or `new reconciliation`. Start the run, provide the source files or source records requested by the workflow, and wait for generated output.

</Step>
<Step title="Review results">

Search **Ask Darpan** for `run result`, `saved result`, `open result`, the run name, or the output file name. Open the result, review differences, and decide whether the issue is source data, mapping, schema selection, or expected business variance.

</Step>
</Steps>
