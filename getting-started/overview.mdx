Darpan helps teams compare data from operational systems, find mismatches, and preserve the evidence needed to investigate them.

## Core workflow

1. Open **Ask Darpan**.
2. Search for the page, workflow, record, or outcome you need.
3. Configure the external connections Darpan needs.
4. Create or select schemas for the source data.
5. Run reconciliation.
6. Review generated output and differences.
7. Adjust source data, schema configuration, or rules when the output shows drift.

## Main surfaces

| Surface | Purpose |
| --- | --- |
| Connections | Saved setup for systems such as SFTP, NetSuite, and AI providers. |
| Schemas | Definitions that describe source file or record structure. |
| Reconciliation | Workflows that compare inputs and generate result output. |
| Results | Saved run evidence, difference views, and downloadable generated files. |
| Admin and settings | Access, company context, and operational configuration. |

## Navigation model

Ask Darpan is the front door for product navigation. Tutorials use search terms such as `SFTP`, `create schema`, `compare files`, and `run result` instead of assuming a fixed sidebar or menu path.

## What Darpan is not

Darpan is not the system of record for source business data. It reads, compares, and reports differences so the owning systems can be corrected or reviewed.

<Tip>
Keep schema names, run names, and connection descriptions specific enough that another user can understand the run history later.
</Tip>
