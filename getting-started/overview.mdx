Darpan is built around one primary outcome: set up a reconciliation run, execute it, and review trustworthy differences.

The app is not a collection of separate settings pages. Connections, schemas, RuleSets, saved runs, results, and Ask Darpan navigation all support the same workflow: make it easy to configure and repeat a reconciliation run.

## Core workflow

1. Sign in and land on the **Dashboard**, the workspace for saved reconciliation runs.
2. Choose an existing run from **Pinned Runs** or **Other Runs** when the setup already exists.
3. Select **Create New** when you need to set up a new reconciliation run.
4. Confirm the reusable setup the run needs: source systems, connection records, schemas, primary ID fields, and rules.
5. Save the reconciliation run so the same comparison can be repeated with new source data.
6. Execute the run from the saved setup or **Run Reconciliation**.
7. Review generated output, difference counts, run history, and result files.
8. Adjust source data, schema configuration, or rules when the output shows drift, then rerun when needed.

## Main surfaces

| Surface | How it helps the run |
| --- | --- |
| Dashboard | Shows saved runs, pinned runs, and the next place to start. |
| Ask Darpan | Lets users navigate through the app and search for pages, workflows, saved runs, setup records, and results. |
| Connections | Stores SFTP, NetSuite, AI provider, and similar setup so users do not re-enter access details for each run. |
| Schemas | Describes source data shapes so run setup can select known fields instead of relying on manual interpretation. |
| Reconciliation runs | Guides users through setting up a new run, selecting sources and schemas, executing the comparison, and returning to saved run history. |
| Results | Lets users view generated output, difference counts, run history, and result files after a reconciliation run. |
| Admin and settings | Controls access, company context, and operational configuration used during runs. |

## Navigation model

Ask Darpan is the front door for product navigation. Search for destinations such as **Open SFTP Servers**, **Create Schema**, **Run Reconciliation**, and **Open Result: ...** to move directly to setup, execution, or review.

## When to use each area

- Use the **Dashboard** when you want to reopen or run a saved reconciliation.
- Use **Create New** when the reconciliation run has not been set up yet.
- Use **Connections** when a run needs access to SFTP, NetSuite, or another external system.
- Use **Schemas** when Darpan needs to understand the fields in a source file or record.
- Use **Results** when you need to inspect what a completed run found.

## What Darpan is not

Darpan is not the system of record for source business data. It reads, compares, and reports differences so the owning systems can be corrected or reviewed.

<Tip>
Keep schema names, run names, and connection descriptions specific enough that another user can understand the run history later.
</Tip>
