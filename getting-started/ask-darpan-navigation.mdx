title: "Ask Darpan navigation"
description: "How to move through Darpan with search-first navigation."
---

Ask Darpan is the primary navigation model in Darpan. Instead of hunting through a permanent sidebar or nested menu, you describe the page, workflow, setting, or record you want to open.

## Open Ask Darpan

Use either entry point:

- Press `Cmd/Ctrl+K`.
- Click the floating **Ask Darpan** button.

Then type the official destination name, workflow name, record name, host, run name, or output file name.

## Select a result

Use the keyboard when you can:

1. Type your search.
2. Use Up/Down to move through results.
3. Press Enter to open the selected result.
4. Press Escape to close Ask Darpan.

When you are inside a workflow, Escape can return you to the surface that launched the workflow.

## What Ask Darpan can find

Ask Darpan searches static app destinations and selected record-backed destinations.

| What you want | Official search term | Result to open |
| --- | --- | --- |
| Main workspace | **Go to Dashboard** | **Go to Dashboard** |
| AI provider settings | **Open AI Settings** | **Open AI Settings** |
| SFTP settings | **Open SFTP Servers** | **Open SFTP Servers** |
| A saved SFTP record | SFTP name, host, or username | **Edit SFTP: ...** |
| NetSuite setup | **Open NetSuite Settings** | **Open NetSuite Settings** |
| Schema library | **Open Schema Library** | **Open Schema Library** |
| Schema creation | **Create Schema** | **Create Schema** |
| Reconciliation setup | **Create Reconciliation Flow** | **Create Reconciliation Flow** |
| Run a comparison | **Run Reconciliation** | **Run Reconciliation** |
| Saved result output | Run name or output file name | **Open Result: ...** |
| Roadmap and requests | **Open Requests and Roadmap** | **Open Requests and Roadmap** |

## Search behavior

Ask Darpan is designed for natural search inside the app. It can match labels, descriptions, aliases, prefixes, and close spelling variations. Documentation should still use official destination names so tutorials remain stable.

<Tip>
If the first query does not find the target, use the official result label shown in the docs or search by the saved record name.
</Tip>

## How tutorials use navigation

Docs should say "open Ask Darpan and search..." with the official destination name when the user needs to move to a Darpan surface. Page-local buttons, cards, tables, or forms are described only after Ask Darpan opens the correct surface.

This keeps tutorials aligned with the product's command-first shell instead of assuming a traditional app menu.
